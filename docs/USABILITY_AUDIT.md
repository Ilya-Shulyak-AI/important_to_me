# Usability Audit

| Issue | Affected screen | Severity | User impact | Planned fix | Implementation status | Verification performed |
|---|---|---:|---|---|---|---|
| Widget screen implied native widgets but exposed Swift/Xcode prototype and simulated controls. | iPhone Widgets | High | Normal users could copy incomplete developer code and think fake settings affected real widgets. | Replace with Scriptable setup, real export, real script, and plain language. | Implemented | `npm run lint`, `npm run test`, `npm run build` attempted/run; source reviewed. |
| Widget export included unnecessary native bridge/localStorage sync. | App shell/widgets | High | Confusing architecture and unsupported PWA widget claims. | Remove automatic bridge sync and export only minimal Scriptable JSON. | Implemented | Source reviewed; Scriptable data omits notes/photos/custom fields. |
| README lacked prominent live app link and current widget behavior. | Documentation | Medium | Users could not easily find the app or understand widget setup. | Add canonical app URL and Scriptable guide. | Implemented | Link checked with network attempt; environment returned 403 for external fetch, so README states current known URL. |
| Old branding used photo/JPEG icon with feminine styling. | App icon, favicon, navigation | Medium | Visual identity felt less neutral and less scalable. | Replace active assets with original neutral SVG identity. | Implemented | Branding test verifies referenced files exist. |
| Relationship, location, and custom field labels required repetitive typing. | People and Events forms | Medium | Slow repeated entry and inconsistent duplicates. | Add reusable searchable creatable combobox. | Implemented | Automated combobox utility tests. |
| Tags could duplicate with different casing. | People and Events forms | Medium | Filter lists become messy. | Prevent case-insensitive duplicates while preserving first-entered casing. | Implemented | Automated utility/source tests. |
| Linked-person selector did not search. | Events form | Medium | Long people lists were hard to scan. | Add search above linked people picker. | Implemented | Source tests verify linked-person search UI exists. |
| Active filters were easy to forget. | People and Events lists | Medium | Users could see empty states without realizing filters were active. | Add visible Clear filters button. | Implemented | Source tests verify clear-filter UI. |
| Month/day precision still showed full date picker. | People and Events forms | Medium | Unknown-year dates were hard to enter accurately. | Use text input for month-day/year precision and keep full date picker for full dates. | Implemented | Source reviewed. |
| Technical and generated labels appeared in normal UI. | Widgets, People, Events, Settings | Low | Labels sounded confusing or overly technical. | Simplify widget naming and several form labels. | Implemented | Source reviewed. |
| Empty states were generic. | People, Events, Widgets | Low | Users lacked next-step guidance. | Add clear messages and actions where practical. | Implemented | Source reviewed. |
| External PR/CI/deployment inspection was limited by repository checkout lacking a remote. | Repository workflow | Medium | Could not inspect live GitHub metadata from local git configuration. | Document limitation and still inspect local branches, commits, workflows, and files. | Implemented | `git branch -a`, `.github/workflows/ci.yml`, `git log`. |
